# 1. 변수(Variable) 기본 개념
- **변수 = 이름 붙인 상자**: 값을 담아 재사용.
- **동적 타이핑 vs 정적 타이핑**  
  - 파이썬: 타입 선언 없이 할당 순간에 변수 생성, 이후 다른 타입도 재할당 가능.  
  - 자바: `int x = 10;`처럼 **타입을 먼저 고정**하고 이후 값이 타입과 맞아야 함.
- 할당 예시  
  ```python
  x = 10          # 정수
  name = "파이썬"  # 문자열
  pi = 3.14
  ```
- 같은 이름에 다른 타입을 넣을 수 있지만, 의미가 바뀌면 새 변수명 사용하기.
- 타입 힌트(선택): 자바처럼 명시하고 싶을 때  
  ```python
  age: int = 20
  nickname: str = "som"
  ```
  실행 시 강제하지 않지만 IDE/검사기(MyPy)로 오류를 미리 잡기 좋음.

# 2. 작명 규칙과 관례
- 소문자 + 밑줄(`snake_case`) 사용: `user_name`, `total_sum`.
- 숫자로 시작 불가, 공백 불가, 예약어( `for`, `class` 등 ) 사용 금지.
- 의미가 드러나게: `score_avg`, `max_speed`.
- 낙타표기법을 피하는 이유: 파이썬 표준 스타일(PEP 8)이 `snake_case`를 권장하며, 표준 라이브러리와 일관되게 맞추면 가독성·팀 협업에 유리하다. 혼용하면 함수/변수 이름을 눈으로 스캔할 때 오히려 헷갈린다.
- 상수 표기: 관례로 모두 대문자 + 밑줄(`MAX_SIZE`, `API_URL`). 언어 차원 상수는 아니므로(재할당 가능) **모듈 상단에 모아두고** 변경하지 않는다는 의도를 주석이나 코드 구조로 드러내기.
- 피해야 할 이름/형태  
  - 예약어: `False`, `None`, `True`, `and`, `as`, `assert`, `async`, `await`, `break`, `class`, `continue`, `def`, `del`, `elif`, `else`, `except`, `finally`, `for`, `from`, `global`, `if`, `import`, `in`, `is`, `lambda`, `nonlocal`, `not`, `or`, `pass`, `raise`, `return`, `try`, `while`, `with`, `yield` 등은 변수명으로 사용할 수 없음.
  - 하이픈(`-`): 뺄셈 연산자로 해석되므로 변수명에 사용할 수 없다. 단어 구분은 밑줄 사용.
  - `PascalCase`/`CamelCase`: 클래스 이름에 예약된 스타일이므로 변수에는 쓰지 않는다.
  - `__dunder__` 패턴: 파이썬 내부 예약 네임스페이스이므로 새로 만들지 않기. 필요 시 단일 밑줄 접두어(`_temp`) 정도로 한정.

# 3. 여러 변수 한 번에 다루기
- **동시 할당**  
  ```python
  a, b, c = 1, 2, 3
  ```
- **값 교환**  
  ```python
  a, b = b, a
  ```
- **별표 언패킹(나머지 모으기)**  
  ```python
  first, *middle, last = [10, 20, 30, 40]
  # first=10, middle=[20, 30], last=40
  ```
- 자바 비교: 자바의 `int a = 1, b = 2;`처럼 한 줄 선언은 비슷하지만, 컬렉션을 바로 나눠 담는 언패킹 문법이 더 유연하다.
